# JSON Wire Rally Integration Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add JSON wire protocol support to Rally, gated behind `protocol = "json"` config, without leaking JSON parsing or protocol details into Rally runtime code.

**Architecture:** A generated `protocol_wire` facade sits between Rally's generated code and Libero's wire modules. For ETF, it delegates to `libero/wire`. For JSON, it delegates to `libero/json/wire` (server) and `libero/json/wire_ffi.mjs` (client). Rally runtime never imports `JSON.parse`, `JSON.stringify`, or raw JSON wire modules directly. The facade is generated per-namespace and selected at codegen time based on the `protocol` field in `ScanConfig`.

**Tech Stack:** Gleam (server generators), JavaScript (client transport), Libero JSON wire modules (codegen, wire, frame, error)

- [ ] **Step 1: Add protocol_wire generator function**

In `src/rally/generator.gleam`, add:

```gleam
pub fn generate_protocol_wire(protocol: String, atoms_module: String, wire_module: String) -> String {
  case protocol {
    "json" -> json_protocol_wire_source(atoms_module, wire_module)
    _ -> etf_protocol_wire_source(atoms_module, wire_module)
  }
}

fn etf_protocol_wire_source(atoms_module: String, wire_module: String) -> String {
  "// Generated by Rally — do not edit.

import gleam/dynamic.{type Dynamic}
import libero/error.{type DecodeError}
import libero/wire as libero_wire

@external(erlang, \"" <> atoms_module <> "\", \"ensure\")
fn ensure_atoms() -> Nil

pub fn encode(value: a) -> BitArray {
  libero_wire.encode(value)
}

pub fn decode_call(data: BitArray) -> Result(#(String, Int, Dynamic), DecodeError) {
  libero_wire.decode_call(data)
}

pub fn encode_call(module module: String, request_id request_id: Int, msg msg: a) -> BitArray {
  libero_wire.encode_call(module:, request_id:, msg:)
}

pub fn encode_request(module module: String, request_id request_id: Int, msg msg: a) -> BitArray {
  libero_wire.encode_request(module:, request_id:, msg:)
}

pub fn encode_response(request_id request_id: Int, value value: a) -> BitArray {
  libero_wire.encode_response(request_id:, value:)
}

pub fn encode_push(module module: String, value value: a) -> BitArray {
  libero_wire.encode_push(module:, value:)
}

pub fn tag_push(module module: String, msg msg: a) -> BitArray {
  libero_wire.tag_push(module:, msg:)
}

pub fn variant_tag(value: Dynamic) -> Result(String, Nil) {
  libero_wire.variant_tag(value)
}

pub fn coerce(value: a) -> b {
  libero_wire.coerce(value)
}

pub fn encode_flags(value: a) -> String {
  libero_wire.encode_flags(value)
}

pub fn decode_flags_typed(flags: String, decoder_name: String) -> Result(a, DecodeError) {
  libero_wire.decode_flags_typed(flags:, decoder_name:)
}
"
}
```

- [ ] **Step 2: Add JSON protocol_wire source**

```gleam
fn json_protocol_wire_source(atoms_module: String, _wire_module: String) -> String {
  "// Generated by Rally — do not edit.

import gleam/dynamic.{type Dynamic}
import gleam/json
import gleam/result
import libero/frame.{type ServerFrame}
import libero/json/error.{type JsonError}
import libero/json/wire as json_wire

@external(erlang, \"" <> atoms_module <> "\", \"ensure\")
fn ensure_atoms() -> Nil

// Contract hash is generated by libero/json/contract and embedded
// in the generated protocol_wire at codegen time. Read from the
// rpc_contract.json output (which Libero always generates) or
// compute from the contract generator result.
const contract_hash = \"GENERATED_AT_CODEGEN_TIME\"

pub fn encode(value: a) -> BitArray {
  panic as \"encode: JSON protocol does not use raw ETF encode\"
}

pub fn decode_call(data: BitArray) -> Result(#(String, Int, Dynamic), DecodeError) {
  panic as \"decode_call: JSON protocol does not use ETF decode_call\"
}

pub fn encode_call(module module: String, request_id request_id: Int, msg msg: a) -> BitArray {
  panic as \"encode_call: JSON protocol does not use ETF encode_call\"
}

pub fn encode_request(module module: String, request_id request_id: Int, msg msg: json.Json) -> String {
  json_wire.encode_request(module:, request_id:, msg:, contract_hash:)
}

pub fn encode_response(request_id request_id: Int, value value: json.Json) -> String {
  json_wire.encode_response(request_id:, value:)
}

pub fn encode_push(module module: String, value value: json.Json) -> String {
  json_wire.encode_push(module:, value:)
}

pub fn tag_push(module module: String, msg msg: a) -> BitArray {
  panic as \"tag_push: JSON protocol uses encode_push\"
}

pub fn variant_tag(value: Dynamic) -> Result(String, Nil) {
  panic as \"variant_tag: JSON protocol uses typed decode\"
}

pub fn coerce(value: a) -> b {
  panic as \"coerce: JSON protocol uses typed decode\"
}

pub fn decode_server_frame(data: String) -> Result(ServerFrame(Dynamic), List(JsonError)) {
  json_wire.decode_server_frame(data)
}

pub fn encode_flags(value: json.Json) -> String {
  json_wire.encode_flags(value)
}

pub fn decode_flags_typed(flags: String, decoder: fn(Dynamic) -> Result(a, List(JsonError))) -> Result(a, List(JsonError)) {
  json_wire.decode_flags_typed(flags, decoder)
}
"
}
```

```gleam
  request_id: Int,
  msg: json.Json,
  contract_hash: String,

pub fn send_rpc(msg: json.Json, callback: fn(b) -> Nil) -> Nil {
  send_raw("/ws", "rpc", msg, callback)

```

The `send_to_server` wrapper pre-encodes the message using the generated JSON codec before calling `send_raw`.

- [ ] **Step 2: Update transport_ffi.mjs for JSON**

Add a JSON-aware `send` function that:
1. Calls the generated JSON encoder for the message
2. Calls `encode_request` from the JSON wire FFI
3. Sends the JSON string over the WebSocket

The WebSocket must send text frames (strings) in JSON mode, not binary frames.

```js
// JSON send path
export function send(url, module, msg, callback) {
  ensureSocket(url);
  const requestId = nextRequestId++;
  const payload = encode_request(module, requestId, msg);
  // ws.send(payload) — payload is a string in JSON mode
}
```

- [ ] **Step 3: Add JSON WebSocket text frame handling**

In `transport_ffi.mjs`, when in JSON mode:
- `ws.send(textPayload)` — sends text frames
- `ws.addEventListener("message", ...)` — receives text frames
- `decode_server_frame(event.data)` — decodes JSON string

Detect JSON mode via a flag passed from generated config.

### Task 4: JSON server request decode and dispatch

**Files:**

- [ ] **Step 1: Generate protocol-branching in WS handler**

The generated WS handler must branch on protocol in the frame handling code:

```gleam
case msg {
  mist.Binary(data) -> {
    // ETF path (unchanged)
    case protocol_wire.decode_call(data) { ... }
  }
  mist.Text(data) -> {
    // JSON path
    case protocol_wire.decode_request(data) {
      Ok(envelope) -> {
        // Validate contract hash
        // Typed-decode message via generated JSON decoder
        // Dispatch normally
      }
      Error(errors) -> {
        // Return JSON error frame
      }
    }
  }
  ...
}
```

- [ ] **Step 2: Handle mist.Text in WS handler**

Mist's `WebsocketMessage` type has a `Text(String)` variant for text frames. The generated handler must handle this when protocol is JSON.

- [ ] **Step 3: Generate JSON error frames on decode failure**

On bad JSON, bad contract hash, bad type, or bad field shape:
- Return a JSON error frame via `protocol_wire.encode_error(...)`
- Do not fall into ETF decode
- Do not crash the socket process

- [ ] **Step 4: Update HTTP handler for JSON**

The HTTP handler needs to detect the protocol mode and decode requests accordingly:
- ETF: binary body via `wire.decode_call(body)`
- JSON: text body via `json_wire.decode_request(body, expected_hash)`

- [ ] **Step 5: Add tests**

Test each error case: bad JSON, bad contract hash, bad type, bad field shape. Verify error frame is returned, socket stays open.
